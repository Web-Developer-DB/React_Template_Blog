## Warum Responsive Design neu gedacht werden muss

Responsive Design ist seit Jahren Standard, doch viele Codebasen nutzen noch immer Breakpoint-Tabellen à la „mobile, tablet, desktop“. Das Problem: Wir optimieren für Gerätekategorien, nicht für Inhalte. Moderne CSS-Features wie Container Queries oder `clamp()` erlauben ein Umdenken. Wir gestalten in Flows, nicht mehr in festen Pixelrastern.

```css
.article-title {
  font-size: clamp(1.75rem, 1.2rem + 1.8vw, 2.8rem);
}
```

Der Code oben zeigt, wie Fluid Typography minimaler CSS bedarf. Wir deklarieren ein Minimum, ein Maximum und eine Wachstumsrate. Browser interpolieren dazwischen. Kein Tooling, keine Berechnung – einfach deklarativ.

## Container Queries in der Praxis

Container Queries lösen das klassische Layout-Dilemma: Ein Card-Grid weiß selbst, wie viel Platz zur Verfügung steht, und kann seine Darstellung unabhängig vom Viewport anpassen.

```css
.card-grid {
  display: grid;
  gap: 1.5rem;
  container-type: inline-size;
}

@container (min-width: 520px) {
  .card-grid {
    grid-template-columns: repeat(2, minmax(0, 1fr));
  }
}
```

Die Kombination aus Container und Query ermöglicht Layout-Logik, die sich an der tatsächlichen Breite des Containers orientiert. Kein Rätselraten mehr, welcher Breakpoint wohl passen könnte.

## Accessibility & Performance nicht vergessen

- `prefers-reduced-motion` steuert Animationen.
- `prefers-contrast` bereitet dich auf bessere Barrierefreiheit vor.
- Responsives Design ist auch Performance: kleinere Assets, adaptive Bilder, schlanke DOM-Strukturen.
